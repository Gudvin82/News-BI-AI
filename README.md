# YouPiter BI — Product Showcase

Демонстрационный репозиторий аналитической платформы для такси-оператора:
- финансы,
- найм,
- маркетинг,
- интеграции (Bitrix24, Google Sheets, Yandex Direct),
- админка Telegram-бота найма.

## Важно

Это **showcase/sandbox версия**, а не production-исходник.

Что сознательно удалено:
- реальные токены и ключи,
- production webhook URL,
- runtime логи и персональные данные,
- закрытая бизнес-логика и внутренние операционные детали.

## Что здесь можно оценить

- архитектуру модулей,
- качество интерфейсов,
- подход к интеграциям и валидации,
- админские UX-потоки (конструкторы, проверки, публикация конфигов),
- базовые API-паттерны и структуру проекта.

## Для быстрого просмотра (5 минут)

1. Откройте `docs/ARCHITECTURE.md` — обзор модулей и потоков данных.
2. Откройте `docs/API_EXAMPLES.md` — как устроены ключевые API-контуры.
3. Откройте `apps/youpiter-bi-sandbox/src/app/(app)/marketing/hire-bot/page.tsx` — пример продуктовой админки-конструктора.
4. Откройте `mock-data/` — безопасные демо-данные для презентаций.

## Структура

- `apps/youpiter-bi-sandbox` — урезанная sandbox-версия BI/админки
- `apps/driver-bot-sandbox` — sandbox-код Telegram-бота найма
- `docs/` — архитектура, ограничения showcase, примеры API
- `mock-data/` — демо-данные для локальной демонстрации

## Быстрый старт (sandbox)

1. Перейдите в `apps/youpiter-bi-sandbox`
2. Скопируйте `.env.example` в `.env.local`
3. Подставьте тестовые (не production) значения
4. Запустите `npm install` и `npm run dev`

## Демо-ограничения

- Данные в sandbox могут быть моками или частично отключенными интеграциями.
- Production SLA/нагрузка/безопасность демонстрируются по запросу отдельно.
- Скриншоты UI можно добавить отдельным пакетом, если нужен формат one-pager для инвестора/работодателя.
